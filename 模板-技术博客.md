# 技术博客编写规范

## 1. 文件头部元数据（Front Matter）

---
title: 文章标题
category: 分类名称
tag: [标签1,标签2]
description: 文章描述
date: YYYY-MM-DD
author: dewyyr
icon: 图标名称（如paw、code、book等，默认paw）
isOriginal: false
sticky: false
timeline: true
article: true
star: false

---

## 2. 文章内容结构

- 简介 ：文章概述和背景介绍
- 架构原理/源码分析 ：技术原理和代码分析
- 可视化图表 ：使用mermaid等工具绘制架构图、流程图、时序图
- 使用场景及代码示例：实际应用场景描述，代码和代码演示
- 注意事项/建议 ：使用时的注意事项
- 常见问题及方案 ：问题排查和解决方案
- 总结 ：文章要点总结

## 3. 文件命名和存储规范

- 路径 ： src/posts/子目录/
- 文件名 ： ##-标题.md （##从01开始递增）
- 更新索引 ：生成后更新同级README.md文件

## 4. 内容质量要求

- 图文结合 ：配合图表说明复杂概念
- 表述严谨 ：使用专业术语，避免模糊语言
- 内容详实 ：提供足够的技术细节和示例
- 避免重复 ：不重复描述相同内容

## 5. 特殊处理规则

- 图表无法绘制时 ：添加TODO说明图表构成和关系
- 可引用外部图片 ：使用标准markdown语法
- 遵循markdownlint规范 ：确保格式规范

## 6. AI执行要点

1. 先确定合适的子目录，检查现有文件序号
2. 创建符合模板结构的markdown文件
3. 生成可视化图表（优先使用mermaid）
4. 更新目录索引文件
5. 确保内容专业、详实、图文并茂
